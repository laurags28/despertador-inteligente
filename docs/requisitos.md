# Requisitos funcionales del sistema

## Funcionalidades básicas

El sistema debe permitir:

* Crear alarmas.
* Eliminar alarmas.
* Activar y desactivar alarmas.
* Configurar hora y minuto.
* Configurar nombre o etiqueta de la alarma.
* Gestionar múltiples alarmas.
* Configurar repetición semanal.
* Configurar sonidos personalizados.
* Configurar volumen.
* Posponer alarmas (snooze).
* Detener alarmas.
* Consultar próximas alarmas activas.

## Funcionalidades avanzadas seleccionadas

El sistema incluirá las siguientes funcionalidades avanzadas:

### 1. Modo vacaciones

Permite desactivar temporalmente todas las alarmas sin eliminarlas.

### 2. Detección de conflictos

El sistema detectará alarmas configuradas con poca diferencia horaria y avisará de posibles conflictos.

### 3. Categorías de alarmas

Las alarmas podrán clasificarse por categorías:

* Trabajo
* Estudio
* Medicina
* Deporte
* Personal

## Restricciones técnicas

* El sistema no tendrá interfaz gráfica.
* Toda la lógica será implementada en Java.
* La interacción se realizará mediante consola usando un programa main.
* El sistema debe seguir principios de orientación a objetos.
* El código deberá estar modularizado y desacoplado.
